## 2. Results

### §7.3(a) — Conformance

**1,000 / 1,000 blueprint-bound sessions conformant (100%).**
Realized length 18–20 (mean 19.99); stop reasons: 995 max_items,
5 SE-threshold (gate open, minimums already met). Per-constraint minima
held in every session — e.g. `unit-03` (required 2): realized 2–5;
`timss=Reasoning` (required floor(0.2·L)=3–4): realized 4–9.

### §7.3(b) — Precision cost

| Measure | Blueprint-bound | Unconstrained |
|---|---|---|
| Final RMSE | **0.2846** | 0.2991 |
| Bias | 0.227 | 0.240 |
| Mean items | 19.99 | 17.31 |
| MAE @ N=3 / 5 / 10 | 0.483 / 0.403 / 0.301 | identical |
| Stopped early via SE | 0.5% | 76.8% |
| Sessions covering all 11 units | **100%** | **3.6%** |

Paired comparison (same seeds): absolute-error difference −0.013 favoring
the constrained arm, p = 3.8×10⁻⁵.

**Reading:** enforcement re-ordered almost nothing — item sequences are
IDENTICAL to pure MI through the shared prefix (verified per-simulee), because
the priority index only intervenes when a minimum is actually threatened.
MAE-at-N is therefore identical by construction at equal N; the entire
"cost" of the blueprint is **+2.7 items of test length** (the §3.4 gate
holding sessions open to finish minimums), which buys a small final-RMSE
*improvement* and — the operative guarantee — **total content coverage**:
unconstrained MI skipped ≥1 curriculum unit entirely in **96.4%** of
sessions (mean coverage 8.89/11 units).

### §7.4 — Tight-pool stress (flag path)

**200 / 200 sessions completed; 0 stranded; 200 / 200 flagged
`blueprint_conformant: false`.** Every conformance record names exactly one
violated constraint — `unit=unit-11: realized 2 (the entire pruned pool),
required 4` — and every *satisfiable* constraint was still served. The
compile-time warning ("minimum but no bank items match" class) fired as
designed; sessions ran to max_items and ended normally per §3.4(2). No
session was aborted for blueprint reasons.

## 3. Operational notes

- **Runtime:** ~10.5 h wall for 2,200 sessions, single RQ worker,
  python-orchestrated (~7–9 s/session at ≤20 items). The R-native batch lane
  cannot host enforcement (it never returns control between items); options
  and recommendation recorded in the session log — parallelize the Python
  lane if constrained simulation becomes routine.
- **Defect found (queue):** `simulations_queue` `default_timeout` = 6 h;
  the stress job was silently dropped after waiting ~9.5 h behind
  experiment A. Re-enqueued with explicit timeout. **Follow-up:** scale the
  job timeout with experiment size and alert on vanished jobs.
- Enforcement overhead: ≤1 extra `rank_items` call per administered item;
  the fast path (engine suggestion accepted) applies once minimums are met.
